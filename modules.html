sys

var sys.argv: List[String]
sys.argv contains all command-line arguments that were passed to the program. When Lily is being run from mod_lily, this is empty.

server

mod_lily provides this module for the scripts that it runs. Scripts that do not run from mod_lily, but wish to emulate it, can create a server.lly file that provides these things.

var server.env: Hash[String, Tainted[String]]

This is contains the environment that the server has installed upon it. Naturally, this varies from server to server. Since this is typed as String to String, any values within the environment that are not utf-8 valid are removed beforehand.

defineserver.escape(input: String):String

This checks self for having "&", "<", or ">". If any are found, then a new String is created where those html entities are replaced (& becomes &amp;, < becomes &lt;, > becomes &gt;).

var server.get: Hash[String, Tainted[String]]

This contains key+value pairs that were sent to the server as GET variables. Any pair that has a key or a value that is not valid utf-8 will not be present.

var server.httpmethod: String

This is the method that was used to make the request to the server. Common values are "GET", and "POST".

var server.post: Hash[String, Tainted[String]]

This contains key+value pairs that were sent to the server as POST variables. Any pair that has a key or a value that is not valid utf-8 will not be present.

define server.write_literal(text: String)

This writes text directly to the server. If text is not a String literal, then ValueError is raised. No escaping is performed on the text sent, as the text is a String literal.

define server.write_raw(text: String)

This writes text directly to the server. No escaping is done upon text. Use this only if you are certain that there is no possibility of html injection.

define server.write(text: String)

This escapes, then writes text to the server. It is equivalent to text |> server.escape |> server.write_raw, except faster because it skips building an intermediate String value.

server

The server module provides a very basic interface to Postgres. This module is available so long as Lily has been built with Postgres support. This module provides two classes: Result and Conn. Both classes have only methods, and are not inheritable.

Result.close(self: Result)

Closes self, and relinquishes any resources it was holding.

Result.each_row(self: Result, fn: Function(List[String]))

This calls fn upon each row that is present within self. If self is empty, then this does not do anything. This function exhausts all rows that are present within the Result

Result.row_count(self: Result):Integer

This function returns the number of rows that are present within self. If self is closed, then this returns 0.

Conn.query(self: Conn, question: String, args: List[String]...):Either[String, Result]

Attempts to perform a query with the self. Any instance of ? within question is replaced with an element within args. If there were too few arguments to replace, or the query failed, then the result is a Left describing the error. If the query succeeded, the result is a Right with the Result object.

Conn.open(host: *String="", port: *String="", dbname: *String="", name: *String="", pass: *String=""):Option[Conn]

Attempts to open a new database connection using the information provided. On success, this yields a Some holding the new database connection. On failure, it yields a None.
